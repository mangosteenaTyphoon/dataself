- **网关地址**: http://localhost:8080

## 一、目标管理接口测试用例

### 1.1 新增目标（POST /goal）

**测试用例 TC-GOAL-001：成功创建草稿目标**

**前置条件**：
- 用户已登录
- 拥有 `todo:goal:add` 权限

**测试步骤**：
```bash
POST http://localhost:8080/todo/goal
Content-Type: application/json
Authorization: Bearer {token}

{
  "title": "2026年第一季度OKR目标",
  "description": "完成产品核心功能开发和上线",
  "status": "DRAFT",
  "priority": "HIGH",
  "expectedStartTime": "2026-01-01 00:00:00",
  "expectedEndTime": "2026-03-31 23:59:59",
  "tags": "产品开发,核心功能"
}
```

**预期结果**：
- HTTP状态码：200
- 返回数据包含：
  ```json
  {
    "code": 200,
    "msg": "操作成功",
    "data": "新增成功"
  }
  ```
- 数据库中新增一条记录，status为DRAFT
- progress默认为0
- score默认为0

**测试数据**：
| 字段 | 值 | 说明 |
|------|-----|------|
| title | 2026年第一季度OKR目标 | 必填 |
| description | 完成产品核心功能开发和上线 | 可选 |
| status | DRAFT | 必填，枚举值 |
| priority | HIGH | 必填，枚举值 |
| expectedStartTime | 2026-01-01 00:00:00 | 必填 |
| expectedEndTime | 2026-03-31 23:59:59 | 必填 |
| tags | 产品开发,核心功能 | 可选 |

### 4.2 目标得分自动计算

**测试用例 TC-LOGIC-003：所有任务完成后自动计算目标得分**

**前置条件**：
- goalId=1的目标有3个任务
- 任务评分分别为：EXCELLENT(95)、GOOD(85)、QUALIFIED(75)
- 目标按时完成

**测试步骤**：
1. 完成最后一个任务
2. 查询目标详情

**预期结果**：
- 任务质量得分 = (95+85+75)/3 = 85
- 时间完成得分 = 90（按时完成）
- 目标得分 = 85*0.7 + 90*0.3 = 86.5

---

**测试用例 TC-LOGIC-004：逾期完成的目标得分计算**

**前置条件**：

**预期结果**：
